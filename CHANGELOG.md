# 更新日志

本文件记录 ImeTool 各版本面向用户的主要变化。

## [1.0.25] - 2026-07-17

### 修复

- 修复 Chromium 浏览器空输入框将占位文本或整个控件矩形误报为光标位置，导致气泡偏到输入框左侧或下方的问题。
- 修复命令面板等使用活动候选项的组合输入框中，焦点落在列表项后无法定位真实文本输入框的问题。
- 修复在命令面板输入文本后，UI Automation 跨到相邻 DOM 元素并使气泡突然飞离光标的问题。
- 过滤与当前输入框没有交集的过期 MSAA/UI Automation 坐标，避免浏览器界面重绘后继续使用旧位置。

### 优化

- 浏览器 MSAA 光标现在会以当前文本输入框为锚点校验并校正垂直位置，使气泡更稳定地显示在光标右下侧。
- 完善空输入框、占位文本、活动候选项、越界字符矩形及浏览器坐标校正的回归测试。

## [1.0.24] - 2026-07-16

### 新增

- 新增“窗口边框提示”，可使用中文、英文和大写状态颜色描绘当前窗口边缘，并支持调整边框宽度。
- 新增“鼠标旁标记”，文本光标捕获失败时可在系统 IBeam 指针附近显示现有标记样式，并支持独立位置偏移。
- 新增“系统文本光标着色”，可按当前输入状态为 IBeam 指针着色；暂停、禁用、状态未知或退出时自动恢复系统指针。
- 常规设置新增统一数据储存位置，默认跟随软件安装目录，集中保存运行日志和持久化窗口记忆。
- 新增 Info、Warn、Error 三级日志捕获，默认使用 Warn；日志窗口支持刷新、复制选中或全部内容，以及导出日志文件。
- 项目采用 MIT 开源协议，安装程序会展示协议并安装 `LICENSE.txt`。

### 优化

- 窗口记忆持久化文件统一保存为数据目录中的 `window-memory.json`，旧版自定义文件会自动迁移到统一位置。
- 日志改为异步分级写入，文件超过 8 MB 时自动轮转，避免频繁诊断信息影响输入状态轮询。
- 高级应用规则区域改为紧凑工具栏、明确空状态和自适应列表高度，减少无规则时的大块空白。
- 移除设置项之间的重复分隔线，并将“设置窗口材质”移动到“开机启动”之后，简化常规页面层级。
- 三种扩展提示方式默认关闭，升级后保持原有光标气泡行为。
- 安装程序会完整检查 x64 `.NET 9 Desktop Runtime`、Core Runtime、hostfxr 和关键 WPF 组件；缺失时使用主备下载源安装并进行 SHA-256 与安装后完整性验证。
- 当运行库安装要求重启时，安装程序会提示重启并暂缓启动 ImeTool，避免依赖尚未就绪时启动失败。

## [1.0.23] - 2026-07-16

### 新增

- 光标捕获模式扩展为自动、Win32 Caret、UI Automation、MSAA、浏览器兼容和 JetBrains/JAB 六种。
- 新增 MSAA `OBJID_CARET` 捕获器，为 Firefox 和部分传统无障碍控件提供光标位置回退。
- 新增浏览器环境识别：Chromium 优先 UI Automation，Firefox 优先 MSAA，并以经过验证的 Win32 Caret 兜底。
- 新增 Java Access Bridge 动态加载，可从 Java、JetBrains 安装目录或目标应用自带 JBR 中查找运行库。

### 优化

- 自动模式会识别标准窗口、Chromium、Firefox 和 Java/JetBrains 环境，按对应捕获链选择结果。
- MSAA 使用独立后台读取线程，避免跨进程无障碍调用阻塞设置窗口和标记刷新。
- 缓存当前前台窗口类型，减少 50ms 轮询期间重复读取进程信息。

## [1.0.22] - 2026-07-16

### 新增

- 设置页增加“光标捕获模式”，支持自动融合、Win32 原生和 UI Automation 三种策略。
- 捕获模式保存后即时切换，可针对传统桌面软件、浏览器或现代应用选择更合适的光标来源。

### 优化

- 保存设置后保持设置窗口打开，并立即应用窗口记忆、输入法识别、标记显示与光标捕获配置。
- 加宽输入法诊断规则按钮，避免高 DPI 或较大字体下文本显示不完整。
- 精简历史版本窗口，移除重复的大标题、副标题和底栏说明。
- 取消按钮改用无圆圈图标，设置窗口材质使用更清晰的图层材质图标。

## [1.0.21] - 2026-07-16

### 新增

- 设置页增加“输入法诊断”，实时显示焦点窗口、键盘布局 HKL、Language ID、Open Status、Conversion Mode、Caret 来源与最终判定来源。
- 支持将当前输入法原始状态数据捕获为中文或英文识别规则，适配状态码表现特殊的输入法。
- 应用规则新增高级单栏编辑器，可按进程、窗口标题、窗口类和控件类组合匹配。
- 应用规则支持独立隐藏标记、禁用窗口记忆、禁止状态恢复，并可为单个应用追加 X/Y 位置偏移。

### 优化

- “检测已打开窗口”同时读取窗口类和焦点控件类，可直接预填高级规则。
- 窗口记忆与状态恢复规则彻底解耦，关闭自动恢复后仍可继续更新该窗口的实时记忆状态。
- 旧版“完全排除”和“不恢复状态”设置会自动迁移并继续生效。

## [1.0.20] - 2026-07-16

### 新增

- 设置页增加“历史版本”入口，可在独立窗口中查看应用内置的全部更新日志。
- 更新历史按版本从新到旧展示版本号、发布日期与详细内容。

### 优化

- 加宽“在 GitHub 查看”按钮，避免高 DPI 或字体缩放下文本显示不完整。
- 更新历史窗口沿用设置窗口的云母或亚克力材质。

## [1.0.19] - 2026-07-16

### 修复

- 移除“按下 Shift 或 Ctrl + Space 就直接推断中英状态已切换”的逻辑，避免系统禁用对应快捷键后气泡仍错误变化。
- 中英气泡现在只响应输入法实际状态码变化，不再由普通按键事件直接取反。

### 优化

- 参考 InputTip 的状态检测思路，同时轮询 IME open status 与 conversion mode，并自动采用当前真正发生变化的状态信号。

## [1.0.18] - 2026-07-16

### 新增

- 设置页的软件更新区域现在会显示最新版本的更新说明与发布时间。
- 更新日志支持标题、列表、链接等常见 Markdown 内容，并可直接打开对应的 GitHub Release 页面。

### 优化

- 更新检测通过 GitHub Releases Atom Feed 获取说明，Feed 暂时不可用时仍可继续完成版本与安装包校验。
- GitHub Release 改为从本文件提取当前版本说明，确保软件内能展示完整、易读的更新内容。

## [1.0.17] - 2026-07-16

### 修复

- 实时中英状态与窗口记忆解耦，窗口记忆不再覆盖气泡的即时状态。
- 阻止同一窗口内部焦点变化重复恢复输入法状态。
- 改善 Shift、Ctrl + Space 切换输入法时的状态刷新时序。
